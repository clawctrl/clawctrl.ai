üìñ Overview

Clawctrl is a self-hosted AI agent orchestration stack. It consists of two components:

OpenClaw Gateway ‚Äî the core AI runtime. Runs locally on your machine, manages agent sessions, connects to AI providers (Anthropic, OpenAI, Gemini, etc.), and listens on messaging channels (Telegram, Discord, Slack, WebChat).

Clawctrl Dashboard ‚Äî a Next.js web UI for monitoring tasks, reviewing agent output, managing projects, and controlling the gateway. Connects to OpenClaw over WebSocket.

Both are free and open source under the MIT license. Source: github.com/clawctrl/clawctrl.ai

üöÄ Quickstart

The fastest path: use this wizard to configure everything, download the ZIP, then run one command.

chmod +x setup.sh && bash setup.sh

The script will: check for Node 22+, install OpenClaw via the official installer, write your openclaw.json config, clone and set up Clawctrl, seed the database, then start both services.

Requirements: Linux, macOS, or Windows WSL2. Node.js 22+ (auto-installed if missing). Git.

After setup, your stack is accessible at:

Clawctrl Dashboard  ‚Üí  http://localhost:4000
OpenClaw Gateway    ‚Üí  ws://127.0.0.1:18789
Validate            ‚Üí  openclaw doctor

Cost tip: Use claude-haiku-4-5 or gemini-3-flash-preview as your default. They handle 90% of tasks at a fraction of frontier model cost. Reserve expensive models for specific agents via per-agent model overrides.

‚ùì FAQ

How do I add a second AI provider after setup?
Edit ~/.openclaw/openclaw.json, add the provider under models.providers, then run openclaw gateway stop && openclaw gateway start.

Can I run Clawctrl on a separate machine from OpenClaw?
Yes. Set gateway.bind to any (or configure Tailscale), then point OPENCLAW_GATEWAY_URL in .env.local to the remote host.

My Gemini model string stopped working.
Google deprecates preview models with ~2 weeks notice. Check ai.google.dev/gemini-api/docs/models for current strings and update models.providers.gemini.defaultModel in your config.

How do I run OpenClaw as a background service?
Use a process manager: pm2 start "openclaw gateway start" --name openclaw or create a systemd unit. See the GitHub repo for example service files.

Where are logs stored?
Gateway logs stream to stdout by default. To persist: openclaw gateway start >> ~/.openclaw/gateway.log 2>&1 &

Is my API key safe?
Keys are stored only in ~/.openclaw/openclaw.json (chmod 600) and .env.local (chmod 600) on your machine. They are never sent to clawctrl.ai ‚Äî the wizard runs entirely in your browser.
